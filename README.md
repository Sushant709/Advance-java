🔹 1. Introduction
Open source software forms the backbone of modern computing. From operating systems to development tools, most technologies rely on collaboratively built and freely shared code. This project explores Git, a distributed version control system, not only as a tool but as a representation of open-source philosophy.

🟢 Part A — Origin and Philosophy
A1. Problem Git Solved
Before Git, developers used centralized version control systems like SVN. These systems had major issues:
Slow performance
Dependency on central server
Poor handling of large projects
In 2005, Linus Torvalds created Git when proprietary tools failed Linux development.
👉 Problem:
Linux kernel development needed a fast, distributed system.
👉 Solution:
Git introduced:
Distributed version control
Speed
Branching flexibility
Git allowed developers to work independently and merge efficiently.
A2. License Analysis (GPL v2)
Git is licensed under GPL v2.
✔ Four Freedoms:
Freedom to run program
Freedom to study source code
Freedom to modify
Freedom to distribute
✔ Git supports all four freedoms.
Yes — BUT:
Must share source code (GPL requirement)
🔁 GPL vs MIT:
Feature	GPL	MIT
Code sharing	Mandatory	Optional
Commercial use	Allowed	Allowed
Restrictions	Strong	Minimal
👉 I would choose MIT for flexibility.
💡 Free vs Freedom:
Free beer = no cost
Freedom = control
Git = freedom, not just free.
A3. Ethics of Open Source
✔ Should all software be open source?

✔ “Standing on shoulders of giants”
Means:
Innovation builds on existing work
👉 Open source accelerates innovation
